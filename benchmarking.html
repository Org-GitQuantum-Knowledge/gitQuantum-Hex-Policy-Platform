Benchmarking Guide

Running Benchmarks

The HEX Policy Platform includes a comprehensive benchmarking suite for performance analysis.

Basic Benchmark

import { BenchmarkRunner } from '@hex/benchmark';

const runner = new BenchmarkRunner({
  iterations: 100000,
  ruleCount: 10,
  warmup: true
});

const result = runner.run();
console.log(BenchmarkRunner.formatResult(result));

Multiple Runs

const results = runner.runMultiple(5);
console.log(BenchmarkRunner.formatMultipleResults(results));

Configuration Options

The BenchmarkRunner accepts a configuration object:

  • iterations: Number of policy evaluations (default: 100000)
  • ruleCount: Number of rules to generate (default: 10)
  • warmup: Enable JIT warmup phase (default: true)

Interpreting Results

Each benchmark result contains:

  • hexTime: Total execution time for HEX VM in milliseconds
  • opaTime: Total execution time for OPA simulator in milliseconds
  • speedup: Ratio of OPA time to HEX time (values > 1 mean HEX is faster)
  • iterations: Number of policy evaluations performed
  • timestamp: When the benchmark was executed

Performance Tips

  1. Warmup Phase: Always enable warmup to stabilize JIT compilation
  2. Multiple Runs: Run benchmarks multiple times to identify variance
  3. System Load: Run benchmarks on a quiet system for consistent results
  4. Rule Optimization: Place frequently matching rules first
  5. Context Complexity: Simpler contexts evaluate faster
