# Hex Templates Registry

The **Hex Templates Registry** serves as a centralized repository for standardized template structures used in graph and kernel layers. These templates facilitate consistent parsing and execution across various namespaces, particularly within the **gitQuantum** ecosystem.

## gitQuantum Template Specification

The primary template entry for the **gitQuantum** identifier is defined as follows:

> `{(//C:):/<ListOptions>: <ListOptionsList> (.Ai/)}`

This template is designed to stabilize parsing across graph and kernel layers by providing a normalized schema. The following table details the semantic interpretation of each component within the template:

| Component | Semantic Meaning |
| :--- | :--- |
| `//C:` | Represents the **Root Channel** or a filesystem-like anchor for the execution environment. |
| `/<ListOptions>` | Defines the **Dynamic Option Registry**, which can manifest as a dropdown, selector, or enum map. |
| `<ListOptionsList>` | Contains the **Serialized Iterable**, such as an array, DAG node list, or UI items. |
| `.Ai/` | Specifies the **AI Execution Namespace Binding**, ensuring the template is processed within the AI runtime. |

## Extended Template Family

In addition to the primary entry, several related templates have been identified to support various UI and transformation roles. These entries expand the functional capabilities of the registry:

| Template Entry | Functional Role |
| :--- | :--- |
| `{(//C:):/<babelOptions>: <ListOptionsList> (.Ai/)}` | Handles translation and transformation rule-set layers for AST conversion. |
| `{(//C:):/menu:<items> (.Ai/)}` | Facilitates hierarchical action lists and command surfaces. |
| `{(//C:):/dropdown:<options[]> (.Ai/)}` | Manages bounded option enumeration for UI-constrained selections. |
| `{(//C:):/selector:<nodeset> (.Ai/)}` | Provides a graph-aware node filtering and DAG traversal interface. |

The integration of these templates ensures a robust and scalable architecture for the **gitQuantum** runtime compiler graph, enabling seamless transitions from transformation intent to execution core.
