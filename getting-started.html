Getting Started with HEX Policy Platform

Installation

The HEX Policy Platform is available as an npm package. Install it using:

npm install @hex/core @hex/benchmark

Quick Start

Here's a simple example to get you started:

import { PolicyVM, compileToBytecode } from '@hex/core';

// Define your policy rules
const rules = [
  {
    attribute: 'role',
    element: 'document',
    perm: 'read'
  }
];

// Compile rules to bytecode
const bytecode = compileToBytecode(rules);

// Create a VM instance
const vm = new PolicyVM(bytecode);

// Execute a policy decision
const decision = vm.execute({
  subject: { role: 'admin' },
  object: { type: 'document' },
  action: 'read'
});

console.log(decision); // 'ALLOW'

Understanding the Architecture

The HEX Policy Platform uses a three-stage pipeline:

  1. AST (Abstract Syntax Tree): Your policy rules are parsed
  2. DAG (Directed Acyclic Graph): Rules are optimized into a dependency graph
  3. Bytecode: The DAG is compiled into efficient bytecode for execution

Core Concepts

Policy Rules

A policy rule consists of three components:

  • attribute: The subject attribute to match
  • element: The object type to match
  • perm: The action/permission to match

Context

The context is the data being evaluated against your policies:

interface Context {
  subject: Record<string, any>;  // User/actor information
  object: Record<string, any>;    // Resource information
  action: string;                  // Action being performed
}

Decision

The result of policy evaluation is always one of two values:

  • ALLOW: The action is permitted
  • DENY: The action is not permitted (default)
